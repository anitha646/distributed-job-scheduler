# API Documentation

Base URL: `http://localhost:8000`. Interactive Swagger UI at `/docs`
(auto-generated by FastAPI from the code — always the source of truth;
this file is a human-readable companion).

All endpoints except `/auth/*`, `/health`, `/workers/register`,
`/workers/{id}/heartbeat`, and `/jobs/claim` require:
`Authorization: Bearer <token>`

## Auth

| Method | Path | Body | Description |
|---|---|---|---|
| POST | `/auth/register` | `{email, password, full_name}` | Create a user |
| POST | `/auth/login` | `{email, password}` | Returns `{access_token, token_type}` |

## Projects & Queues

| Method | Path | Description |
|---|---|---|
| POST | `/projects` | Create a project under an organization |
| GET | `/projects` | List projects |
| POST | `/queues` | Create a queue (priority, concurrency limit, retry strategy) |
| GET | `/queues?project_id=` | List queues, optionally filtered |
| POST | `/queues/{id}/pause` | Pause a queue (no new claims) |
| POST | `/queues/{id}/resume` | Resume a paused queue |
| GET | `/queues/{id}/stats` | Job counts by status for this queue |

## Jobs

| Method | Path | Description |
|---|---|---|
| POST | `/jobs` | Create a job. `run_at` in the future → delayed/scheduled. `cron_expression` → recurring template. |
| POST | `/jobs/batch` | Create many jobs under one `batch_id` |
| GET | `/jobs?queue_id=&status=&page=&page_size=` | Paginated, filterable job list |
| GET | `/jobs/{id}` | Job detail including executions and logs |
| POST | `/jobs/{id}/retry` | Manually re-queue a failed/dead-lettered job |

### Job status values
`queued → scheduled → claimed → running → completed`
On failure: `running → failed → (queued for retry) | dead_letter`

## Worker-facing endpoints

| Method | Path | Description |
|---|---|---|
| POST | `/workers/register?hostname=` | Register a new worker process |
| POST | `/workers/{id}/heartbeat?active_jobs=` | Liveness signal |
| POST | `/jobs/claim` | `{worker_id, queue_id?, limit}` — atomically claims up to `limit` jobs |
| POST | `/jobs/{id}/start?worker_id=` | Marks job running, opens a `job_executions` row |
| POST | `/jobs/{id}/complete?execution_id=` | `{status, result?, error_message?}` — finalizes the attempt |
| GET | `/workers` | List all workers and their status |

## Dashboard

| Method | Path | Description |
|---|---|---|
| GET | `/dashboard/summary` | Job counts by status + count of online workers |

## Error format

FastAPI's default validation errors (422) and explicit `HTTPException`
calls (401/404/400) return:
```json
{ "detail": "human-readable message" }
```

## Example: submitting a delayed job

```bash
curl -X POST localhost:8000/jobs \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{
    "queue_id": "‹queue-uuid›",
    "job_type": "generate_report",
    "payload": {"report_type": "weekly_summary"},
    "run_at": "2026-07-03T09:00:00Z",
    "max_attempts": 5
  }'
```
